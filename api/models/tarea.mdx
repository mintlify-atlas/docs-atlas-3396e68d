## Overview

The `Tarea` model represents a task that can be assigned to household members. It includes features for task completion, assignment, history tracking, and peer ratings.

## Properties

<ResponseField name="id" type="string" optional>
  Unique identifier for the task. Auto-generated by Firestore.
</ResponseField>

<ResponseField name="nombre" type="string" required>
  Name or title of the task.
</ResponseField>

<ResponseField name="descripcion" type="string" optional>
  Detailed description of what the task involves.
</ResponseField>

<ResponseField name="completada" type="boolean" required>
  Indicates whether the task has been completed.
</ResponseField>

<ResponseField name="asignadA" type="string | null" optional>
  User ID of the person assigned to this task, or null if unassigned.
</ResponseField>

<ResponseField name="hogarId" type="string" required>
  ID of the household this task belongs to.
</ResponseField>

<ResponseField name="createdAt" type="Timestamp | FieldValue" optional>
  Firebase Timestamp when the task was created.
</ResponseField>

<ResponseField name="asignadoNombre" type="string | null" optional>
  Display name of the assigned user.
</ResponseField>

<ResponseField name="asignadoFotoURL" type="string | null" optional>
  Profile photo URL of the assigned user.
</ResponseField>

<ResponseField name="peso" type="number" optional>
  Weight or difficulty level of the task, affects points calculation.
</ResponseField>

<ResponseField name="personalizada" type="boolean" optional>
  Indicates if this is a custom task created by a user.
</ResponseField>

<ResponseField name="creadorUid" type="string" required>
  User ID of the person who created this task.
</ResponseField>

<ResponseField name="creadorNombre" type="string | null" optional>
  Display name of the task creator.
</ResponseField>

<ResponseField name="creadorFotoURL" type="string | null" optional>
  Profile photo URL of the task creator.
</ResponseField>

<ResponseField name="historial" type="array" optional>
  Array of historical completion records for this task.
  
  Each history entry contains:
  - `uid` (string): User ID who completed the task
  - `nombre` (string): User's display name
  - `fotoURL` (string, optional): User's photo URL
  - `fecha` (string): Completion date
  - `completada` (boolean): Completion status
  - `puntosOtorgados` (number, optional): Points awarded
  - `pesoUsado` (number, optional): Weight used for calculation
  - `puntuacionFinal` (number, optional): Final score
  - `fechaOtorgados` (string, optional): Date points were awarded
  - `hogarId` (string, optional): Associated household ID
</ResponseField>

<ResponseField name="valoraciones" type="array" optional>
  Array of peer ratings for this task.
  
  Each rating contains:
  - `uid` (string): User ID who rated
  - `puntos` (number): Rating score
  - `comentario` (string, optional): Optional comment
  - `fecha` (string): Rating date
</ResponseField>

<ResponseField name="valoracionesPendientes" type="string[]" optional>
  Array of user IDs who have pending ratings for this task.
</ResponseField>

<ResponseField name="bloqueadaHastaValoracion" type="boolean" optional>
  If true, task is blocked until all pending ratings are completed.
</ResponseField>

## TareaDTO

Extended version of Tarea with guaranteed assigned user information.

```typescript
interface TareaDTO extends Tarea {
  asignadoNombre: string;
  asignadoFotoURL?: string | null;
}
```

```typescript

const newTarea: Tarea = {
  nombre: 'Sacar la basura',
```
