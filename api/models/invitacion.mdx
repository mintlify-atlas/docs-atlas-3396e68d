## Overview

The `Invitacion` model represents an invitation sent to a user to join a household. Each invitation has a unique code and tracks whether it has been used.

## Properties

<ResponseField name="id" type="string" optional>
  Unique identifier for the invitation. Auto-generated by Firestore.
</ResponseField>

<ResponseField name="hogarId" type="string" required>
  ID of the household that the invitation is for.
</ResponseField>

<ResponseField name="email" type="string" required>
  Email address of the person being invited.
</ResponseField>

<ResponseField name="codigo" type="string" required>
  Unique invitation code used to accept the invitation.
</ResponseField>

<ResponseField name="creadoEn" type="any" required>
  Timestamp when the invitation was created. Typically a Firebase Timestamp.
</ResponseField>

<ResponseField name="usado" type="boolean" required>
  Indicates whether the invitation has been used/accepted.
</ResponseField>

## Example

```json
{
  "id": "inv_abc123",
  "hogarId": "hogar_456def",
  "email": "nuevo.miembro@example.com",
  "codigo": "ABCD1234",
  "creadoEn": "2024-01-20T10:30:00Z",
  "usado": false
}
```

## Usage

```typescript
import { Invitacion } from '@/app/core/invitaciones/models/invitacion.model';

const newInvitacion: Invitacion = {
  hogarId: 'hogar_123',
  email: 'amigo@example.com',
  codigo: 'XYZ789',
  creadoEn: new Date(),
  usado: false
};
```

## Notes

- Each invitation code should be unique across the system
- Once an invitation is used (`usado: true`), it cannot be used again
- Invitations are typically sent by the household owner or admin
- The `email` field is used to verify that the correct person is accepting the invitation
