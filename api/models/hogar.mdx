## Overview

The `Hogar` model represents a household where multiple users can collaborate on tasks. Each household has an owner, members, and optional configuration like type and ZenPrime subscription status.

## Type Definitions

### TipoHogar

```typescript
type TipoHogar = 'Familiar' | 'Pareja' | 'Amigos' | 'Erasmus'
```

Defines the type of household configuration.

## Properties

<ResponseField name="id" type="string" optional>
  Unique identifier for the household. Auto-generated by Firestore.
</ResponseField>

<ResponseField name="nombre" type="string" required>
  Name of the household.
</ResponseField>

<ResponseField name="countryCode" type="string" required>
  ISO country code for the household's location.
</ResponseField>

<ResponseField name="provincia" type="string" required>
  Province or state name where the household is located.
</ResponseField>

<ResponseField name="provinciaCode" type="string" required>
  Province or state code.
</ResponseField>

<ResponseField name="ownerUid" type="string" required>
  User ID of the household owner who created it.
</ResponseField>

<ResponseField name="miembros" type="string[]" required>
  Array of user IDs representing all members in the household.
</ResponseField>

<ResponseField name="adminUid" type="string" required>
  User ID of the household administrator with special permissions.
</ResponseField>

<ResponseField name="createdAt" type="Timestamp | null" required>
  Firebase Timestamp when the household was created.
</ResponseField>

<ResponseField name="tipoHogar" type="TipoHogar" optional>
  Type of household: 'Familiar', 'Pareja', 'Amigos', or 'Erasmus'.
</ResponseField>

<ResponseField name="zenPrime" type="boolean" optional>
  Indicates if the household has ZenPrime premium subscription.
</ResponseField>
