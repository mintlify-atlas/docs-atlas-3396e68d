## Overview

The `Canjeo` interface represents a record of a reward redemption. Each time a user spends points on a reward, a `Canjeo` document is created in their redemption history.

**Location**: `src/app/core/tienda/models/canjeo.model.ts:1`

## Type Definition

```typescript
export interface Canjeo {
  id: string;
  recompensaId: string;
  titulo: string;
  puntosGastados: number;
  fecha: Date;
  tipo: 'predefinida' | 'personalizada';
}
```

## Fields

<ResponseField name="id" type="string" required>
  Unique identifier for this redemption record. Auto-generated by Firestore.
</ResponseField>

<ResponseField name="recompensaId" type="string" required>
  The ID of the `Recompensa` that was redeemed. References the original reward.
</ResponseField>

<ResponseField name="titulo" type="string" required>
  Snapshot of the reward's title at the time of redemption. Preserved even if the original reward is later modified or deleted.
</ResponseField>

<ResponseField name="puntosGastados" type="number" required>
  Number of points spent on this redemption. Typically matches the reward's `coste` field at the time of redemption.
</ResponseField>

<ResponseField name="fecha" type="Date" required>
  Timestamp when the reward was redeemed. Used for sorting redemption history.
</ResponseField>

<ResponseField name="tipo" type="'predefinida' | 'personalizada'" required>
  Type of reward that was redeemed:
  - `'predefinida'` - System-provided reward
  - `'personalizada'` - Custom household reward
</ResponseField>

## Usage in TypeScript

### Viewing Redemption History

```typescript
import { Firestore, collection, query, orderBy, getDocs } from '@angular/fire/firestore';
import { Canjeo } from './core/tienda/models/canjeo.model';

async function getUserRedemptionHistory(userId: string): Promise<Canjeo[]> {
  const canjeosRef = collection(firestore, `usuarios/${userId}/canjeos`);
  const q = query(canjeosRef, orderBy('fecha', 'desc'));
  const snapshot = await getDocs(q);
  
  return snapshot.docs.map(doc => ({
    id: doc.id,
    ...doc.data()
  })) as Canjeo[];
}
```

### Calculating Total Points Spent

```typescript
function calculateTotalPointsSpent(canjeos: Canjeo[]): number {
  return canjeos.reduce((total, canje) => total + canje.puntosGastados, 0);
}

// Example usage
const history = await getUserRedemptionHistory('user-abc');
const totalSpent = calculateTotalPointsSpent(history);
console.log(`Total points spent: ${totalSpent}`);
```

### Filtering by Type

```typescript
function getCustomRewardRedemptions(canjeos: Canjeo[]): Canjeo[] {
  return canjeos.filter(c => c.tipo === 'personalizada');
}

function getPredefinedRewardRedemptions(canjeos: Canjeo[]): Canjeo[] {
  return canjeos.filter(c => c.tipo === 'predefinida');
}
```

### Recent Activity

```typescript
function getRecentRedemptions(canjeos: Canjeo[], days: number = 30): Canjeo[] {
  const cutoffDate = new Date();
  cutoffDate.setDate(cutoffDate.getDate() - days);
  
  return canjeos.filter(c => new Date(c.fecha) >= cutoffDate);
}

// Get last 7 days of redemptions
const recent = getRecentRedemptions(history, 7);
console.log(`${recent.length} redemptions in the last week`);
```

## Firestore Storage

### Collection Path

Redemption records are stored in a subcollection under each user:

```
usuarios/{userId}/canjeos/{canjeoId}
```

### Creation

`Canjeo` documents are created automatically by `TiendaService.canjearRecompensa()`:

```typescript
// Inside canjearRecompensa transaction
const canjeoRef = doc(canjeosRef);
tx.set(canjeoRef, {
  recompensaId: recompensa.id,
  titulo: recompensa.titulo,
  puntosGastados,
  tipo: recompensa.tipo,
  fecha: new Date()
});
```

### Security Rules

Recommended Firestore security rules:

```javascript
match /usuarios/{userId}/canjeos/{canjeoId} {
  // Users can only read their own redemption history
  allow read: if request.auth.uid == userId;
  
  // Only the TiendaService can create redemptions (server-side)
  // In practice, authenticated users create via service
  allow create: if request.auth.uid == userId;
  
  // Redemptions are immutable - cannot be updated or deleted
  allow update, delete: if false;
}
```

<Note>
  Redemption records are intentionally immutable. Once created, they cannot be modified or deleted, ensuring an accurate audit trail.
</Note>

## Why Snapshot the Titulo?

The `titulo` field stores the reward's title at redemption time rather than referencing it dynamically. This is intentional:

<AccordionGroup>
  <Accordion title="Preserves History">
    If an admin renames a reward later, redemption history still shows the original name.
  </Accordion>
  
  <Accordion title="Handles Deletions">
    If a custom reward is deleted, the redemption history remains intact with the reward name.
  </Accordion>
  
  <Accordion title="Audit Trail">
    Creates an accurate, unchangeable record of what was redeemed and when.
  </Accordion>
</AccordionGroup>

## Analytics Use Cases

### Most Popular Rewards

```typescript
function getMostRedeemedReward(canjeos: Canjeo[]): string {
  const counts = new Map<string, number>();
  
  canjeos.forEach(c => {
    counts.set(c.titulo, (counts.get(c.titulo) || 0) + 1);
  });
  
  let maxCount = 0;
  let mostPopular = '';
  
  counts.forEach((count, titulo) => {
    if (count > maxCount) {
      maxCount = count;
      mostPopular = titulo;
    }
  });
  
  return mostPopular;
}
```

### Spending Trends

```typescript
interface MonthlySpending {
  month: string;
  points: number;
  count: number;
}

function getMonthlySpending(canjeos: Canjeo[]): MonthlySpending[] {
  const byMonth = new Map<string, { points: number; count: number }>();
  
  canjeos.forEach(c => {
    const date = new Date(c.fecha);
    const monthKey = `${date.getFullYear()}-${String(date.getMonth() + 1).padStart(2, '0')}`;
    
    const existing = byMonth.get(monthKey) || { points: 0, count: 0 };
    byMonth.set(monthKey, {
      points: existing.points + c.puntosGastados,
      count: existing.count + 1
    });
  });
  
  return Array.from(byMonth.entries()).map(([month, data]) => ({
    month,
    ...data
  }));
}
```

## Integration with Notifications

When a reward is redeemed, the notification system is notified:

```typescript
await notificacionesService.crearNotificacionCanje(
  hogarId,
  usuarioId,
  canjeo.titulo,
  canjeo.puntosGastados
);
```

This creates a household notification so other members can see redemption activity. See [NotificacionesService](/api/notificaciones-service) for details.
