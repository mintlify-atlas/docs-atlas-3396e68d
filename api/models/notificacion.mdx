## Overview

The `NotificacionDTO` model represents notifications sent to household members about various events such as reward redemptions, task acceptances, and rejections.

## Type Definitions

### NotificacionTipo

```typescript
type NotificacionTipo = 'canje' | 'aceptada' | 'rechazo'
```

Defines the type of notification:
- `canje`: A reward has been redeemed
- `aceptada`: A request has been accepted
- `rechazo`: A request has been rejected

## Properties

<ResponseField name="id" type="string" optional>
  Unique identifier for the notification. Auto-generated by Firestore.
</ResponseField>

<ResponseField name="tipo" type="NotificacionTipo" required>
  Type of notification: 'canje', 'aceptada', or 'rechazo'.
</ResponseField>

<ResponseField name="hogarId" type="string" required>
  ID of the household this notification belongs to.
</ResponseField>

<ResponseField name="createdAt" type="any" required>
  Timestamp when the notification was created. Typically a Firebase Timestamp.
</ResponseField>

<ResponseField name="actorUid" type="string" required>
  User ID of the person who triggered the notification.
</ResponseField>

<ResponseField name="actorNombre" type="string" required>
  Display name of the person who triggered the notification.
</ResponseField>

<ResponseField name="recompensaId" type="string" optional>
  ID of the reward associated with this notification (for 'canje' type).
</ResponseField>

<ResponseField name="recompensaTitulo" type="string" optional>
  Title of the reward associated with this notification.
</ResponseField>

<ResponseField name="coste" type="number" optional>
  Cost in points of the reward that was redeemed.
</ResponseField>

<ResponseField name="vistoPor" type="Record<string, true>" optional>
  Object mapping user IDs to true for users who have seen this notification.
  Used to track which household members have viewed the notification.
</ResponseField>

## Usage

```typescript
import { NotificacionDTO, NotificacionTipo } from '@/app/core/hogar/notificaciones/model/notificacion.model';

const newNotificacion: NotificacionDTO = {
  tipo: 'canje',
  hogarId: 'hogar_123',
  createdAt: new Date(),
  actorUid: 'user_456',
  actorNombre: 'Pedro Sánchez',
  recompensaId: 'reward_789',
  recompensaTitulo: 'Día libre de tareas',
  coste: 100,
  vistoPor: {}
};
```

## Notes

- The `vistoPor` field uses a Record type for efficient lookups in Firestore
- Notifications are typically broadcast to all household members except the actor
- The `recompensaId`, `recompensaTitulo`, and `coste` fields are only relevant for 'canje' type notifications
- Notifications can be marked as seen by adding the user's ID to the `vistoPor` object
