# Deployment

ZenHogar is deployed to Firebase Hosting, providing a fast, secure, and globally distributed hosting solution for the Angular application.

## Prerequisites

Before deploying, ensure you have:

<Steps>
  <Step title="Firebase CLI installed">
    ```bash
    npm install -g firebase-tools
    ```
  </Step>
  
  <Step title="Firebase project created">
    - Go to [Firebase Console](https://console.firebase.google.com/)
    - Create a new project or use existing
    - Enable Authentication and Firestore
  </Step>
  
  <Step title="Environment configured">
    ```bash
    # Copy example environment file
    cp src/environments/environment.example.ts src/environments/environment.ts
    cp src/environments/environment.prod.example.ts src/environments/environment.prod.ts
    ```
    
    Fill in your Firebase credentials from Project Settings
  </Step>
  
  <Step title="Firebase CLI authenticated">
    ```bash
    firebase login
    ```
  </Step>
</Steps>

## Firebase Configuration

### Project Setup

Initialize Firebase in your project (if not already done):

```bash
firebase init
```

<Note>
If `.firebaserc` and `firebase.json` already exist, you can skip this step.
</Note>

Select:
- **Hosting** (use space to select)
- Choose existing project or create new
- Public directory: `dist/zenhogar/browser`
- Configure as single-page app: **Yes**
- Set up automatic builds: **No**

### Firebase Project Configuration

The `.firebaserc` file links your local project to Firebase:

```json .firebaserc
{
  "projects": {
    "default": "zenhogar"
  }
}
```

To switch projects:

```bash
firebase use --add
```

### Hosting Configuration

```json firebase.json
{
  "hosting": {
    "public": "dist/zenhogar/browser",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ],
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  }
}
```

<Accordion title="Understanding the configuration">
- **public:** Output directory from Angular build (matches `outputPath` in `angular.json`)
- **ignore:** Files to exclude from deployment
- **rewrites:** Redirect all routes to `index.html` for Angular routing to work
</Accordion>

## Build Process

### Development Build

For testing locally with production settings:

```bash
ng build --configuration development
```

Output: `dist/zenhogar/browser/`

### Production Build

```bash
ng build
# or explicitly
ng build --configuration production
```

This creates an optimized build with:
- ✅ Minification
- ✅ Tree shaking
- ✅ AOT compilation
- ✅ Output hashing for cache busting

### Angular Build Configuration

From `angular.json`:

```json
"configurations": {
  "production": {
    "budgets": [
      {
        "type": "initial",
        "maximumWarning": "2MB",
        "maximumError": "3MB"
      },
      {
        "type": "anyComponentStyle",
        "maximumWarning": "25kB",
        "maximumError": "35kB"
      }
    ],
    "outputHashing": "all"
  },
  "development": {
    "optimization": false,
    "extractLicenses": false,
    "sourceMap": true
  }
}
```

<Warning>
If your build exceeds the budget, you'll get errors. Consider:
- Lazy loading modules
- Removing unused dependencies
- Optimizing images and assets
</Warning>

## Deployment

### Deploy to Firebase Hosting

<Steps>
  <Step title="Build the application">
    ```bash
    npm run build
    ```
    
    Verify `dist/zenhogar/browser/` was created
  </Step>
  
  <Step title="Deploy to Firebase">
    ```bash
    firebase deploy --only hosting
    ```
    
    This uploads the built files to Firebase Hosting
  </Step>
  
  <Step title="Verify deployment">
    Firebase CLI will output a hosting URL:
    
    ```
    ✔  Deploy complete!
    
    Hosting URL: https://zenhogar.web.app
    ```
    
    Visit the URL to confirm deployment
  </Step>
</Steps>

### Complete Deployment Script

Create a deployment script for convenience:

```bash deploy.sh
#!/bin/bash

# Exit on error
set -e

echo "🏗️  Building ZenHogar for production..."
npm run build

echo "🚀 Deploying to Firebase Hosting..."
firebase deploy --only hosting

echo "✅ Deployment complete!"
```

Make it executable:

```bash
chmod +x deploy.sh
./deploy.sh
```

## Firestore Security Rules

Before deploying, set up Firestore security rules:

<Steps>
  <Step title="Create firestore.rules file">
    ```javascript firestore.rules
    rules_version = '2';
    service cloud.firestore {
      match /databases/{database}/documents {
        
        // Helper functions
        function isAuthenticated() {
          return request.auth != null;
        }
        
        function isOwner(uid) {
          return isAuthenticated() && request.auth.uid == uid;
        }
        
        function isMemberOfHome(hogarId) {
          return isAuthenticated() && 
            request.auth.uid in get(/databases/$(database)/documents/hogares/$(hogarId)).data.miembros;
        }
        
        // Users collection
        match /usuarios/{uid} {
          allow read: if isAuthenticated();
          allow write: if isOwner(uid);
          
          // User's home history
          match /historialHogares/{hogarId} {
            allow read: if isOwner(uid);
            allow write: if false; // System only
          }
        }
        
        // Homes collection
        match /hogares/{hogarId} {
          allow read: if isAuthenticated() && 
            request.auth.uid in resource.data.miembros;
          
          allow create: if isAuthenticated();
          
          allow update, delete: if isAuthenticated() && 
            request.auth.uid == resource.data.ownerUid;
          
          // Home subcollections
          match /miembros/{uid} {
            allow read: if isMemberOfHome(hogarId);
            allow write: if isAuthenticated() && 
              request.auth.uid == get(/databases/$(database)/documents/hogares/$(hogarId)).data.ownerUid;
          }
          
          match /eventos/{eventId} {
            allow read: if isMemberOfHome(hogarId);
            allow write: if false; // System only
          }
          
          match /notificaciones/{notifId} {
            allow read: if isMemberOfHome(hogarId);
            allow create: if isMemberOfHome(hogarId);
            allow update, delete: if false;
          }
          
          match /recompensasPersonalizadas/{recompensaId} {
            allow read: if isMemberOfHome(hogarId);
            allow write: if isAuthenticated() && 
              request.auth.uid == get(/databases/$(database)/documents/hogares/$(hogarId)).data.ownerUid;
          }
        }
        
        // Tasks collection
        match /tareas/{tareaId} {
          allow read, write: if isAuthenticated() && 
            isMemberOfHome(resource.data.hogarId);
        }
        
        // Invitations
        match /invitaciones/{invitacionId} {
          allow read: if isAuthenticated();
          allow create: if isAuthenticated();
          allow update, delete: if isAuthenticated() && 
            request.auth.uid == resource.data.creadorUid;
        }
        
        // Assignment requests
        match /peticionesAsignacion/{peticionId} {
          allow read, write: if isAuthenticated() && 
            isMemberOfHome(resource.data.hogarId);
        }
      }
    }
    ```
  </Step>
  
  <Step title="Deploy security rules">
    ```bash
    firebase deploy --only firestore:rules
    ```
  </Step>
  
  <Step title="Test rules in Firebase Console">
    - Go to Firestore → Rules
    - Use the Rules Playground to test scenarios
  </Step>
</Steps>

## Firebase Authentication Configuration

<Steps>
  <Step title="Enable Google Sign-In">
    1. Go to Firebase Console → Authentication → Sign-in method
    2. Enable **Google** provider
    3. Add authorized domains (your hosting domain)
  </Step>
  
  <Step title="Configure authorized domains">
    Add:
    - `localhost` (for development)
    - `your-project.web.app`
    - `your-project.firebaseapp.com`
    - Custom domain (if any)
  </Step>
</Steps>

## Custom Domain (Optional)

<Steps>
  <Step title="Add custom domain in Firebase Console">
    - Go to Hosting → Add custom domain
    - Enter your domain (e.g., `zenhogar.com`)
  </Step>
  
  <Step title="Update DNS records">
    Follow Firebase instructions to add DNS records to your domain provider
  </Step>
  
  <Step title="Wait for SSL certificate">
    Firebase automatically provisions an SSL certificate (can take up to 24 hours)
  </Step>
</Steps>

## Continuous Deployment (CI/CD)

### GitHub Actions Example

Create `.github/workflows/deploy.yml`:

```yaml .github/workflows/deploy.yml
name: Deploy to Firebase Hosting

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Build
        run: npm run build
      
      - name: Deploy to Firebase
        uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: '${{ secrets.GITHUB_TOKEN }}'
          firebaseServiceAccount: '${{ secrets.FIREBASE_SERVICE_ACCOUNT }}'
          channelId: live
          projectId: zenhogar
```

<Note>
You'll need to add `FIREBASE_SERVICE_ACCOUNT` secret to your GitHub repository settings.

Generate it from Firebase Console → Project Settings → Service Accounts.
</Note>

## Deployment Checklist

Before deploying to production:

- [ ] Update `environment.prod.ts` with production Firebase credentials
- [ ] Remove console.log statements or debug code
- [ ] Test build locally: `ng build && firebase serve`
- [ ] Verify Firestore security rules are deployed
- [ ] Verify Firebase Authentication is configured
- [ ] Check Angular build budgets (no errors)
- [ ] Test on multiple browsers
- [ ] Run `npm run lint` to check for code issues
- [ ] Update `package.json` version if needed
- [ ] Commit all changes before deploying

## Rollback

If a deployment has issues:

```bash
# View deployment history
firebase hosting:channel:list

# Rollback to previous version
firebase hosting:clone SOURCE_SITE_ID:SOURCE_CHANNEL_ID TARGET_SITE_ID:live
```

Or use Firebase Console → Hosting → Release history → Rollback

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build fails with budget errors">
    **Solution:** Optimize bundle size or increase budgets in `angular.json`
    
    ```json
    "budgets": [
      {
        "type": "initial",
        "maximumWarning": "3MB",  // Increased
        "maximumError": "4MB"
      }
    ]
    ```
  </Accordion>
  
  <Accordion title="Routes return 404 on refresh">
    **Cause:** Missing rewrite rule in `firebase.json`
    
    **Solution:** Ensure rewrites are configured:
    ```json
    "rewrites": [
      { "source": "**", "destination": "/index.html" }
    ]
    ```
  </Accordion>
  
  <Accordion title="Firebase Auth not working">
    **Causes:**
    - Authorized domains not configured
    - Wrong environment credentials
    - Google provider not enabled
    
    **Solution:** Check Firebase Console → Authentication → Settings
  </Accordion>
  
  <Accordion title="Firestore permission denied errors">
    **Cause:** Security rules too restrictive or not deployed
    
    **Solution:** 
    1. Check rules in Firebase Console
    2. Deploy rules: `firebase deploy --only firestore:rules`
    3. Use Rules Playground to test
  </Accordion>
</AccordionGroup>
